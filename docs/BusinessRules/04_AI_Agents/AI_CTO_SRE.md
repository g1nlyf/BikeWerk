# AI-CTO/SRE - надежность платформы и данных

Статус: Role blueprint  
Дата: 2026-02-15  
Круг: Reliability Circle

## 1. Миссия роли

AI-CTO/SRE защищает техническую основу бизнеса:

- доступность сервисов;
- целостность данных;
- устойчивость бизнес-процессов при сбоях.

## 2. Что делает AI-CTO/SRE

1. Мониторит API latency/error rate и критичные endpoints.
2. Контролирует контракты событий воронки и качество метрик.
3. Выявляет деградации интеграций (CRM/Telegram/платежка).
4. Запускает incident playbooks и контролирует MTTR.
5. Передает технический риск в AI-CEO и смежные круги.

## 3. Связь с другими агентами

## 3.1 Входы

- от AI-РОП: симптомы операционных сбоев.
- от AI-CRO/Traffic: провалы воронки и аномалии атрибуции.
- от AI-CEO: приоритеты критичных бизнес-сервисов.

## 3.2 Выходы

- для AI-РОП: подтверждение техпричин задержек и ETA восстановления.
- для AI-CRO: исключение ложных продуктовых выводов при техдеградации.
- для AI-CEO: incident summary и risk forecast.
- для AI-GRC: сигналы нарушений по логированию и audit trail.

## 4. KPI AI-CTO/SRE

- uptime критичных сервисов;
- API p95/p99;
- incident count;
- MTTR;
- data contract violation count.

## 5. Как тестировать AI-CTO/SRE

## 5.1 Unit

1. Деградационный скоринг сервисов.
2. Детектор аномалий latency/error spikes.
3. Валидатор контракта funnel events.

## 5.2 Integration

1. Реакция на отказ Telegram callback/notifications.
2. Реакция на задержки CRM status writes.
3. Корректная публикация incident сигналов в общий контур.

## 5.3 E2E

1. Частичный outage API: AI-CTO/SRE корректно эскалирует и ставит workaround.
2. Потеря части funnel events: запускается data recovery plan.
3. Деградация интеграции менеджер-бота: AI-РОП получает технический контекст и корректирует SLA.
