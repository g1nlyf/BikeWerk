# AI-РОМ - руководитель операционной модели менеджеров

Статус: Role blueprint  
Дата: 2026-02-15  
Круг: Sales Operations Circle

## 1. Миссия роли

AI-РОМ отвечает за качество исполнения менеджеров:

- стандартизация действий;
- минимизация ошибок и пропусков;
- рост производительности команды без потери качества.

AI-РОМ не заменяет AI-РОП, а усиливает его:
- AI-РОП управляет потоком заказов;
- AI-РОМ управляет качеством людей в этом потоке.

## 2. Что делает AI-РОМ

1. Ведет персональные scorecards менеджеров.
2. Выявляет типовые ошибки и узкие места по сотрудникам.
3. Формирует next best action для менеджера по заказу.
4. Контролирует полноту чеклистов и заметок.
5. Подсвечивает потребность в обучении/перераспределении нагрузки.

## 3. Связь с другими агентами

## 3.1 Входы

- от AI-РОП: SLA breach, queue incidents, проблемные заказы.
- от AI-Head of Quality: дефекты из-за менеджерских ошибок.
- от AI-CFO: случаи потери маржи из-за операционных действий.
- от AI-CEO: приоритеты и ограничения периода.

## 3.2 Выходы

- для AI-РОП: рекомендации по назначению и перераспределению.
- для AI-CEO: кадровые риски и план улучшения команды.
- для AI-Head of Quality: превентивные меры по проблемным паттернам.

## 4. KPI AI-РОМ

- throughput per manager;
- error rate по статусам и чеклистам;
- процент заказов с полной документацией;
- доля повторных ошибок на сотрудника;
- время обучения до стабильного SLA.

## 5. Как тестировать AI-РОМ

## 5.1 Unit

1. Расчет scorecard по менеджеру.
2. Детектор повторяющихся ошибок.
3. Рекомендатор next best action по типу сделки.

## 5.2 Integration

1. Сверка scorecards с фактическими статусами в CRM.
2. Передача рекомендаций AI-РОП и фиксация применения.
3. Проверка полноты данных по чеклистам/заметкам.

## 5.3 E2E

1. Менеджер системно срывает SLA: AI-РОМ выявляет и дает план коррекции.
2. Команда перегружена: AI-РОМ предлагает перераспределение и фиксирует эффект.
3. Рост ошибок по новому этапу: AI-РОМ инициирует стандарт процесса и контролирует улучшение.
